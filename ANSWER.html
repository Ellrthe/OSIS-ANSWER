Ответы на экзаменационные вопросы

Дисциплина «Операционные системы» · составлено по конспекту

1Понятие операционной системы, основные функции и назначение

ОС — комплекс программ, обеспечивающих взаимодействие всех аппаратных и программных частей компьютера между собой, а также взаимодействие пользователя с компьютером. Проще говоря, ОС — это «посредник» между железом и программами.

Функции: управление памятью; управление процессами; работа с файловыми системами; стандартизованный доступ к устройствам; управление доступом к данным; обеспечение пользовательского интерфейса; безопасность; сетевое взаимодействие.

Назначение: обслуживание аппаратуры компьютера; создание рабочей среды и интерфейса для пользователя; выполнение команд; организация ввода-вывода и хранения информации.

ОС относится к системному базовому ПО. Компоненты ОС: загрузчик, ядро, интерфейс, драйверы. Требования к ОС: расширяемость, переносимость, совместимость, надёжность, производительность.

2История развития операционных систем

  • До 1946 г. — механические счётные устройства (абак, Паскалина, арифмометр, разностная машина Бэббиджа).
  • 1946 — ENIAC (первый электронный компьютер). ОС отсутствовали, программисты работали напрямую с физическими адресами.
  • 1960-е — появились диспетчеры — упрощённые ОС для пакетной обработки заданий; появилось мультипрограммирование и разделение времени.
  • 1970 — UNIX — первая переносимая (мобильная) ОС, не зависящая от конкретного железа.
  • 1980-е — ОС для персональных компьютеров: CP/M (8-разрядные), MS-DOS (16-разрядные).
  • 1984 — MacOS с графическим интерфейсом; 1985 — первые графические Windows.
  • Сегодня — Windows доминирует на ПК, Linux — на серверах и мобильных устройствах (Android), наиболее распространены мобильные ОС.

3Основные понятия ОС: системные вызовы, прерывания, исключительные ситуации, файлы, процессы

Системный вызов — интерфейс между прикладной программой и ОС. Программа «просит» ОС выполнить привилегированное действие (создать файл, запустить процесс и т.д.). При этом программа переходит в режим ядра (kernel mode). Иногда называют программным прерыванием.

Прерывание — сигнал процессору о том, что произошло какое-то событие (нажата клавиша, завершилась операция ввода-вывода и т.д.). Процессор приостанавливает текущую работу, выполняет обработчик прерывания (ISR), затем возвращается к прерванной программе. Бывают: аппаратные (от устройств), программные (команда в коде), внутренние / исключения, прерывания по таймеру.

Исключительная ситуация (exception) — событие внутри процессора при невозможности выполнить команду до конца: нет прав доступа, обращение к несуществующей странице памяти и т.п.

Файл — логически взаимосвязанная совокупность информации во внешней памяти, имеющая имя. Файл — именованная область логического адресного пространства.

Файловая система (ФС) — способ организации и хранения файлов на носителях. Определяет правила именования, структуру, размер, атрибуты файлов. Главная цель ФС — скрыть от программиста физические особенности носителя.

Процесс — программа в состоянии выполнения. ОС рассматривает процесс как заявку на потребление всех ресурсов кроме процессорного времени (его получают потоки).

4Системы реального времени

СРВ (Система реального времени) — система, которая обязана реагировать на события внешней среды в рамках строго заданных временных ограничений.

Жёсткое реальное время
Задержка недопустима — это фатальная ошибка. Примеры: бортовые системы управления самолётом, системы аварийной защиты. При нарушении срока ОС блокирует операцию.
Мягкое реальное время
Задержка — восстановимая ошибка. Снижается производительность, но данные не теряются. Пример: потоковая передача видео, сетевые пакеты.

5Организация хранения данных на носителях. Драйверы устройств. Разделы на дисках, дисковые массивы

Диск разбит на секторы (512 байт). Несколько секторов = кластер (минимальная единица распределения для ОС). Раздел — смежная область диска, логически как отдельный диск.

Дисковые массивы (RAID) — несколько дисков, работающих как единое хранилище для надёжности и/или производительности.

Драйвер устройства — программный модуль, позволяющий ОС работать с конкретным устройством. Устройство = механический компонент + контроллер (электронный компонент). Контроллер — интерфейс между устройством и драйвером, имеет локальный буфер. Современные ОС поддерживают динамическую загрузку драйверов — без перекомпиляции ядра.

6Файловые системы: примеры, функции и назначение

Файловая система — способ организации файлов на носителе. Задаёт правила именования, структуру каталогов, атрибуты, способ хранения.

Функции ОС по управлению файлами: создание и удаление, открытие и закрытие, управление директориями, отображение файлов во внешнюю память, резервное копирование.

Примеры ФС: FAT, FAT32 (простые, для небольших дисков и флешек), NTFS (Windows, надёжная, поддерживает большие файлы), ext4 (Linux), APFS (macOS). Linux поддерживает разные ФС через виртуальную файловую систему (VFS).

7Файловая система FAT

FAT (File Allocation Table — таблица размещения файлов) — простая ФС для небольших дисков.

Структура:

  • Загрузочный сектор — первый сектор диска, содержит DPB (параметры диска) и загрузчик.
  • Таблицы FAT1 и FAT2 (две копии для надёжности) — хранят состояние каждого кластера: свободен, занят файлом, плохой, последний в цепочке файла.
  • Корневой каталог — записи о файлах и папках корня.
  • Область данных — сами файлы.

Файл хранится цепочкой кластеров. Каждый кластер содержит ссылку на следующий. Атрибуты файла в FAT: archive, system, hidden, read-only.

Простой язык: FAT — это как «оглавление книги», где для каждой «страницы» (кластера) записано, где искать следующую страницу того же файла.

8Файловая система NTFS, её особенности

NTFS — основная ФС Windows, разработана для быстрого выполнения операций чтения/записи/поиска и восстановления на больших дисках.

Структура тома:

  • Первые 12% — зона MFT (резервируется для метафайла MFT, чтобы он не фрагментировался).
  • Оставшиеся 88% — обычное пространство для файлов.

MFT (Master File Table) — специальный метафайл, определяющий местонахождение всех остальных файлов. Его копия хранится в середине диска для надёжности.

Каждый файл и каталог в NTFS — набор атрибутов (имя, данные, информация защиты и др.). Имя файла — до 256 символов, любые символы. Поддерживаются права доступа, шифрование, сжатие, журналирование.

9Прерывания, функции прерываний в работе ОС

Прерывание — сигнал процессору о наступлении события. Процессор прерывает текущую программу, передаёт управление обработчику прерывания, после чего возвращается к прерванной программе.

Типы прерываний:

  • Аппаратные — от периферийных устройств (нажата клавиша, операция ввода-вывода завершена).
  • Внутренние (исключения) — возникают в самом процессоре (ошибка деления, нарушение прав доступа).
  • Программные — специальная команда в коде, используется для системных вызовов (переход в режим ядра).
  • По таймеру — периодические, для опроса устройств и реализации разделения времени.

Вектор прерываний — таблица, где каждому номеру прерывания соответствует адрес его обработчика. ОС по сути является программой, управляемой прерываниями.

10Организация ввода-вывода. Контроллеры устройств. Драйверы, динамическая загрузка

Ввод-вывод — управление передачей данных между ЦПУ и периферийными устройствами. Устройства бывают: блок-ориентированные (диск — хранит блоки фиксированного размера с адресами) и байт-ориентированные (клавиатура, принтер — поток байт без адресации).

Контроллер — электронный компонент устройства. Работает как интерфейс между устройством и драйвером. Имеет локальный буфер. При оптимизации (режим DMA) буфер — часть оперативной памяти, передача данных идёт без участия процессора.

Драйвер — программный модуль ОС для работы с конкретным устройством. Динамическая загрузка — драйвер подключается/отключается без перезагрузки системы (альтернатива — перекомпиляция ядра, что неудобно).

ОС функции: передавать устройствам команды, перехватывать прерывания, обрабатывать ошибки, обеспечивать интерфейс между устройствами и остальной системой.

11Синхронный и асинхронный ввод-вывод. Буферы. Кэширование. Менеджеры ввода-вывода

Синхронный ввод-вывод — программа запускает операцию и ждёт её завершения, после чего продолжает работу. Привычный способ для программиста.

Асинхронный ввод-вывод — программа запускает операцию и сразу продолжает работу, не ожидая её завершения. Эффективнее при параллельных задачах.

Аналогия: синхронный — сам отнёс письмо и ждёшь ответа стоя у почты; асинхронный — отдал письмо курьеру и пошёл по своим делам.

Буфер — промежуточная область памяти для хранения данных в процессе передачи. Сглаживает разницу в скорости между устройством и процессором.

Кэширование — накопление часто используемых данных в быстром хранилище. Кэш ОЗУ сокращает обращения к медленному диску. Виртуальная память — тоже разновидность кэширования.

Менеджер ввода-вывода — компонент ОС, координирующий все компоненты подсистемы ввода-вывода и взаимодействие с процессами.

12Интерфейс операционной системы. Виды интерфейсов

Интерфейс — «проводник» между человеком и компьютером / программой.

  • Командный интерфейс (CLI) — текстовые команды с клавиатуры или из скрипт-файла. Управление — только клавиатура. Используется в серверных и текстовых ОС (MS-DOS, bash в Linux).
  • Графический интерфейс (GUI) — экранные элементы: иконки, окна, меню. Управление — мышь + клавиатура. Стартовый экран — рабочий стол. Используется в Windows, macOS, современных дистрибутивах Linux.

Интерфейс между программами и ОС — системные вызовы. Между ОС и железом — драйверы и контроллеры.

13Архитектура ОС. Ядро и вспомогательные модули. Загружаемые модули ядра

Ядро — центральная часть ОС, управляет процессами и ресурсами, работает в привилегированном режиме. Вокруг ядра — вспомогательные модули (утилиты, драйверы, библиотеки).

Загружаемые модули ядра (LKM) — фрагменты кода, которые подключаются и отключаются от ядра на лету, без перезагрузки. Расширяют функциональность ядра (например, добавляют поддержку нового оборудования).

При установке ОС происходит её генерация — из набора типовых модулей собирается конкретная ОС для конкретного компьютера.

14Виды ядер ОС

Типы: монолитное, модульное, микроядро, гибридное, экзоядро, наноядро.

15Монолитное ядро ОС

Все компоненты ОС — части одной большой программы, работают в одном адресном пространстве в привилегированном режиме. Каждая процедура может вызвать любую другую.

➕ Плюсы: высокая скорость работы, богатые возможности, поддержка большого числа устройств.

➖ Минусы: сбой одного компонента может обрушить всю систему (всё в одном пространстве). Старые версии требовали перекомпиляции при смене железа.

Примеры: ранние версии Linux, MS-DOS.

16Модульное ядро ОС

Усовершенствованное монолитное ядро. Поддерживает динамическую подгрузку модулей (драйверов, файловых систем и т.д.) без перекомпиляции и перезагрузки. Модули загружаются в адресное пространство ядра и работают как его часть.

Модульность — на уровне бинарных образов, а не архитектуры, поэтому ядро остаётся технически монолитным.

➕ Плюсы: гибкость, не нужно перекомпилировать ядро.

Пример: современный Linux (поддерживает insmod/rmmod для управления модулями).

17Микроядро ОС

Минимальное ядро, работающее в привилегированном режиме: только базовое управление процессами, прерываниями, памятью и IPC. Всё остальное (драйверы, файловые системы) — пользовательские процессы-серверы, общающиеся через передачу сообщений.

➕ Плюсы: надёжность (падение сервиса не обрушивает систему), безопасность, переносимость.

➖ Минусы: накладные расходы на передачу сообщений между компонентами — медленнее монолита.

По принципу микроядра построены некоторые ОС Windows NT и QNX.

18Гибридное ядро ОС

Модифицированное микроядро, которое для ускорения переносит часть «несущественных» сервисов в пространство ядра. Драйверы можно добавлять и в ядро, и в пользовательское пространство — компромисс между скоростью и надёжностью.

Примеры: Windows NT/10/11, macOS (XNU).

19Экзоядро, наноядро

Экзоядро — ядро, предоставляющее только функции взаимодействия процессов и безопасного выделения/освобождения ресурсов. API для приложений предоставляют внешние библиотеки. Цель — дать приложениям прямой доступ к железу для максимальной эффективности (например, СУБД работает с диском на уровне секторов, не файлов).

Наноядро — крайне минималистичное ядро, выполняющее единственную задачу — обработку аппаратных прерываний. Результаты передаёт вышестоящему ПО тоже через механизм прерываний. Всё остальное — вне ядра.

20Аппаратная зависимость и переносимость ОС. Совместимость приложений

Аппаратная зависимость — ОС написана под конкретную архитектуру процессора/железа и не работает на другом. Пример: MS-DOS для x86.

Переносимость (portability) — ОС можно перенести на другую платформу с минимальными изменениями. UNIX стал первой переносимой ОС (1970). Достигается написанием ОС на языке высокого уровня (C) с изоляцией машинно-зависимого кода.

Совместимость приложений — возможность запускать программы, написанные для другой ОС или другой версии ОС. Достигается через слои совместимости, виртуальные машины, эмуляторы.

Требования к ОС: расширяемость, переносимость, совместимость, надёжность, производительность.

21Синхронизация процессов. Методы синхронизации

Синхронизация — организация выполнения параллельных процессов в определённом порядке, чтобы они не мешали друг другу при доступе к общим ресурсам.

Критическая секция (область) — участок кода, где происходит обращение к общим данным. Правило: два процесса не могут одновременно находиться в критической секции.

Взаимное исключение (mutex) — механизм, гарантирующий, что критическую секцию выполняет максимум один процесс.

Тупик (deadlock) — циклическая блокировка, когда процессы ждут ресурсы друг друга бесконечно.

Методы синхронизации:

  • Семафор — системная переменная с операциями «открыть/закрыть». В закрытом состоянии процессы встают в очередь.
  • Монитор — многовходовый модуль с общими данными и операциями над ними. В каждый момент в нём работает не более одного процесса. Надёжнее семафора.
  • Алгоритм банкира (Дейкстра) — предотвращает тупики, проверяя безопасность состояния системы перед каждым выделением ресурса.

22Контекст процесса. Одноразовые и многоразовые операции. Переключение контекста

Дескриптор процесса — структура данных ОС с общей информацией о процессе: PID, состояние, приоритет, владелец, статистика времени.

Контекст процесса — всё, что нужно, чтобы возобновить выполнение процесса с прерванного места: значения регистров процессора, указатель текущей команды, открытые файлы, указатели на области памяти, системные флаги.

Переключение контекста — сохранение контекста текущего процесса и восстановление контекста следующего. Время переключения — скрытая активность (латентность) диспетчера, которую нужно минимизировать.

Одноразовые операции: создание и завершение процесса (выполняются один раз, меняют количество процессов в системе, связаны с выделением/освобождением ресурсов).

Многоразовые операции: приостановка, запуск, блокирование, разблокирование, изменение приоритета — не меняют количество процессов.

23Потоки и процессы

Процесс — программа в выполнении. Имеет собственное адресное пространство (код, данные, стек). ОС изолирует процессы друг от друга — прямой доступ к чужой памяти закрыт. Процесс получает уникальный PID.

Поток (thread) — «нить выполнения» внутри процесса. Несколько потоков одного процесса разделяют его адресное пространство, но имеют собственные регистры и стек. Процессорное время ОС распределяет между потоками, а не процессами.

Зачем потоки?

  • Создать поток ~в 100 раз быстрее, чем процесс.
  • Потоки разделяют память — проще обмениваться данными.
  • Параллельно: один поток считает, другой ждёт ввода-вывода — программа не простаивает.

В UNIX иерархия процессов строгая: каждый новый процесс создаётся через fork() и является дочерним. Прародитель всех — процесс init.

24Диспетчеризация процессов. Стратегии диспетчеризации. Диаграмма Ганта

Диспетчеризация — распределение квантов процессорного времени между процессами. Цель — максимальная загрузка ЦПУ.

Планировщик — выбирает, какой процесс получит ЦПУ. Диспетчер — выполняет само переключение (переключение контекста).

Критерии: использование ЦПУ (↑), пропускная способность (↑), среднее время обработки (↓), среднее время ожидания (↓), время ответа (↓).

Диаграмма Ганта — временна́я шкала с именами процессов и диапазонами их выполнения. Используется для наглядной иллюстрации стратегий диспетчеризации.

Стратегии бывают с прерыванием (вытесняющие) и без прерывания (невытесняющие) текущего процесса.

25Диспетчеризация FCFS (обслуживание в порядке поступления)

First-Come-First-Served (FCFS) — процессы обслуживаются строго в порядке поступления, независимо от их длины.

Пример: Процессы P1(24), P2(3), P3(3) в таком порядке:
Время ожидания: P1=0, P2=24, P3=27. Среднее = (0+24+27)/3 = 17.

Если сначала идут короткие: P2(3), P3(3), P1(24):
Время ожидания: P1=6, P2=0, P3=3. Среднее = (6+0+3)/3 = 3.

Эффект сопровождения (convoy effect) — если первым стоит длинный процесс, короткие процессы долго ждут. Главный недостаток FCFS.

26Диспетчеризация SJF без прерывания (самое короткое задание первым)

Shortest-Job-First (SJF) без прерывания — первым получает ЦПУ процесс с наименьшим временем выполнения. Текущий процесс не прерывается, пока не завершится.

Пример: P1(7, появился в 0), P2(4, в 2), P3(1, в 4), P4(4, в 5):
Порядок: P1→P3→P2→P4. Среднее время ожидания = (0+6+3+7)/4 = 4.

SJF оптимален по среднему времени ожидания для заданного набора процессов.

27Диспетчеризация SJF с прерыванием (SRTF)

Shortest-Remaining-Time-First (SRTF) — вариант SJF с прерыванием. Если появляется новый процесс с более коротким оставшимся временем, чем у текущего — текущий прерывается.

В примере с теми же процессами: в момент t=2 приходит P2(4) — P1 прерывается (осталось 5>4); в t=4 приходит P3(1) — P2 прерывается.

SRTF даёт ещё меньшее среднее время ожидания, чем SJF без прерывания, но требует постоянного отслеживания оставшегося времени.

28Диспетчеризация Round Robin (круговая стратегия)

Round Robin (RR) — все процессы по очереди получают одинаковый квант времени q. После истечения кванта процесс уходит в конец очереди.

При n процессах и кванте q — ни один процесс не ждёт больше (n−1)×q.

  • Если q слишком мало — накладные расходы на переключения контекста становятся ощутимыми.
  • Если q велико — вырождается в FCFS.

RR обеспечивает лучшее время ответа (отзывчивость), чем SJF, но хуже по среднему времени оборота. Идеален для интерактивных систем.

29Управление памятью. Функции ОС. Иерархия памяти

Функции ОС по управлению памятью: отслеживание занятой/свободной памяти; выделение памяти процессам и её освобождение; вытеснение неактивных процессов на диск (swapping) и возврат их обратно; настройка адресов программы на физическую память.

Иерархия памяти (от быстрой к медленной):

  • Регистры процессора → Кэш L1/L2/L3 → Оперативная память (RAM) → Внешняя память (диск, SSD) → Архивные носители.

Чем ниже в иерархии, тем больше объём, ниже скорость, выше надёжность. Для ускорения обращения к медленной памяти используется кэш из более быстрой.

Защита памяти: привилегированный и непривилегированный режимы; регистры базы и границы ограничивают область памяти пользовательской программы.

30Управление оперативной памятью

ОС хранит список свободных и занятых областей. При запросе на выделение памяти применяются стратегии:

  • Первый подходящий — первый свободный участок не меньше нужного.
  • Наиболее подходящий — наименьший подходящий участок (меньше остаток).
  • Наименее подходящий — наибольший участок (снижает фрагментацию больших блоков).

Фрагментация: внешняя — есть общая свободная память, но она не непрерывна; внутренняя — выделяется чуть больше, чем нужно (из-за страниц). Лечится компактировкой (сдвиг занятых областей).

Откачка/подкачка (swapping) — если памяти не хватает, неактивный процесс выгружается на диск, при активации загружается обратно.

31Страничная организация памяти

Физическая и логическая память делятся на блоки одинакового размера — страницы (логические) и фреймы (физические). Размер — степень двойки.

Физические фреймы для страниц процесса могут располагаться в произвольных местах — нет необходимости в непрерывной памяти!

Таблица страниц — отображает номер логической страницы в номер физического фрейма. Хранится в памяти; адрес таблицы — в регистре таблицы страниц.

Логический адрес = (номер страницы p, смещение d). По p находим фрейм в таблице, прибавляем смещение d — получаем физический адрес.

Для ускорения используется ассоциативная память TLB (кэш для таблицы страниц). Бит valid/invalid — страница в памяти или нет.

Аналогия: книга разбита на страницы одинакового размера. Страницы одной главы могут лежать в разных местах библиотеки — таблица страниц подскажет, где каждая.

32Сегментная организация памяти

Программа делится на сегменты переменной длины, каждый с логическим смыслом: код, данные, стек, массив и т.д.

Логический адрес = (номер сегмента s, смещение d). Таблица сегментов содержит base (начальный адрес) и limit (длину) каждого сегмента.

При обращении: проверяем d < limit → физический адрес = base + d. Если d ≥ limit — прерывание (ошибка).

Защита: для каждого сегмента — биты прав (чтение, запись, исполнение). Разные процессы могут разделять сегмент под одним номером.

Недостаток: внешняя фрагментация (переменные длины сегментов).

33Организация виртуальной памяти. Стратегии замещения страниц. Thrashing

Виртуальная память — логическая память процесса отделена от физической и может превышать её объём. Реально используемые страницы хранятся в ОЗУ, остальные — на диске.

При обращении к страницы, которой нет в ОЗУ — отказ страницы (page fault). ОС загружает нужную страницу. Если свободных фреймов нет — выбирает «жертву» по алгоритму замещения.

Алгоритмы замещения страниц:

  • FIFO — вытесняется страница, загруженная раньше всех.
  • LRU (Least Recently Used) — вытесняется страница, к которой дольше всего не обращались.
  • Оптимальный (OPT) — вытесняется страница, которая не понадобится дольше всего в будущем (теоретически оптимален, практически нереализуем).
  • LFU/MFU — вытесняется наименее/наиболее часто используемая страница.

Thrashing (пробуксовка) — катастрофическая ситуация: процессор почти всё время тратит на загрузку/выгрузку страниц, а не на полезную работу. Причина — слишком мало фреймов. Лечение — управление рабочими множествами (держать в ОЗУ набор страниц, активно используемых каждым процессом).

34Операционные системы для работы в сети. Сетевые ОС

Сетевая ОС — ОС, поддерживающая работу в компьютерной сети: доступ к удалённым файлам, принтерам и другим ресурсам.

Узлы сети высоко автономны — каждый работает независимо. Пользователь явно указывает, к какому ресурсу хочет подключиться. Легко добавить/удалить машину.

Функции: управление файлами и каталогами в сети, управление ресурсами, коммуникация, защита, отказоустойчивость.

Структура: средства управления локальными ресурсами + серверная часть (предоставление ресурсов другим) + клиентская часть (редиректор — перенаправляет запросы к удалённым ресурсам).

35Распределённые операционные системы

Распределённая ОС — создаёт иллюзию единого компьютера: единое адресное пространство, единая файловая система, единый механизм IPC — независимо от того, что узлы физически разные.

Сетевая ОС
Локальные и сетевые ресурсы — отдельно. Пользователь знает о сети. Управление локальное.
Распределённая ОС
Единое пространство ресурсов. Сеть прозрачна. Управление глобальное. Высокая автоматизация.

Примеры применения: суперкомпьютеры, банки с множеством филиалов, роботизированные заводы.

36Клиентские и серверные операционные системы

Серверные ОС — специализированы для управления серверами: работа с большими объёмами данных, множество одновременных пользователей, управление сетью (DHCP, DNS, Active Directory). Администрирование преимущественно через командную строку. Примеры: Windows Server, Linux.

Клиентские ОС — ориентированы на одного пользователя, удобный GUI, поддержка разнообразного железа, мультимедиа. Примеры: Windows 10/11, macOS, Ubuntu Desktop.

В клиент-серверных системах серверы предоставляют: файл-серверы, серверы приложений, баз данных, веб-серверы, почтовые серверы и т.д.

37Кластеры серверов. Виды кластеров

Кластер — группа серверов, работающих как единая система для повышения производительности и надёжности.

Виды:

  • Симметричные — все узлы равнозначны, работают параллельно.
  • Асимметричные — один узел активен, другие в режиме горячего резерва.
  • С балансировкой нагрузки — запросы распределяются между узлами для равномерной загрузки.
  • С высокоскоростным доступом — используются для вычислений, требующих быстрого обмена данными (HPC — High Performance Computing).

38Протоколы. Стек протоколов TCP/IP

Протокол — набор правил обмена данными между устройствами в сети.

Стек TCP/IP — набор протоколов Интернета (4 уровня):

  • Прикладной уровень — HTTP, FTP, SMTP, DNS (протоколы приложений).
  • Транспортный уровень — TCP (надёжная доставка с подтверждением) и UDP (быстрая доставка без гарантий).
  • Сетевой уровень — IP (адресация и маршрутизация пакетов).
  • Уровень сетевого доступа — Ethernet, Wi-Fi (физическая передача).

TCP гарантирует доставку всех данных (используется для HTTP, email). UDP быстрее, но ненадёжен (используется для видеостриминга, игр, DNS).

39Доменная система имён (DNS). WINS

DNS (Domain Name System) — служба преобразования доменных имён (например, google.com) в IP-адреса. Без DNS пришлось бы везде вводить числовые адреса.

Структура иерархическая: корневые серверы → серверы доменов верхнего уровня (.com, .ru) → серверы конкретных доменов.

Чтобы добавить сервер в Интернет — достаточно дать ему сетевой адрес и зарегистрировать символическое имя в DNS.

WINS (Windows Internet Name Service) — служба Microsoft для преобразования NetBIOS-имён в IP-адреса. Аналог DNS, но для локальных сетей Windows (исторически использовалась в корпоративных сетях до повсеместного перехода на DNS).

40Основные сетевые службы. WWW- и FTP-серверы

WWW (World Wide Web) — система веб-страниц, передаваемых по протоколу HTTP/HTTPS. Веб-сервер хранит и отдаёт HTML-страницы по запросу браузера.

FTP (File Transfer Protocol) — протокол для передачи файлов между компьютерами. FTP-сервер позволяет скачивать и загружать файлы. Используется для распространения ПО, хранения архивов.

Другие основные службы: почтовые серверы (SMTP/IMAP), DNS, DHCP (автоматическая раздача IP-адресов), серверы баз данных, прокси-серверы.

41Основные характеристики и особенности ОС семейства Windows

  • Развитие от графической оболочки для MS-DOS (Windows 1.0, 1985) до полноценной ОС.
  • Наиболее популярная ОС для настольных и портативных ПК.
  • Архитектура ядра — гибридное (NT-ядро).
  • Поддерживает FAT и NTFS (основная ФС — NTFS).
  • Многозадачность, многопользовательский режим, поддержка широкого набора устройств.
  • Богатый GUI (рабочий стол, проводник), но есть и командная строка (CMD, PowerShell).
  • Реестр — централизованное хранилище настроек ОС и приложений.
  • Облачная интеграция через Windows Azure, OneDrive.

42ОС Windows. Интерфейсы. Командные скрипты

Windows поддерживает GUI и командную строку (CMD) и PowerShell.

Командная строка (CMD): dir — список файлов, cd — смена каталога, md/rd — создать/удалить папку, copy/move/del — работа с файлами.

PowerShell: мощнее CMD, командлеты (Get-Process, Where-Object, Sort-Object). Позволяет фильтровать и сортировать процессы, получать детальную информацию.

Командные скрипты (.bat файлы): набор команд в текстовом файле. Поддерживают: переменные (%DATE%, %UserName%), условия (if exist), циклы, метки (:метка, goto), параметры вызова (%0–%9), запуск программ (start, call).

43Инструментарий управления в Windows

Управление процессами в CMD: tasklist — список процессов; taskkill /PID <n> — завершить процесс; tasklist /v /fo list — подробная информация.

Управление пользователями: net user (добавление, удаление, настройка аккаунтов); net localgroup — управление группами (например, добавить в администраторы: net localgroup Администраторы NewUser /add).

PowerShell: Get-Process, управление службами (Get-Service), сетью (Get-NetAdapter), файлами, реестром. Инструментарий WMI/CIM для глубокого управления системой.

44Основные характеристики и особенности ОС Linux

  • Открытый исходный код (Open Source), бесплатная.
  • Ядро Linux (монолитное модульное). Первая версия — 1991 г., Линус Торвальдс.
  • Основана на концепциях UNIX.
  • Многопользовательская, многозадачная.
  • Существует в виде множества дистрибутивов (Ubuntu, Debian, CentOS, Arch и др.).
  • Доминирует на серверах (около 90% серверов), суперкомпьютерах; используется в Android (ядро).
  • Поддерживает разные ФС через VFS: ext4, btrfs, xfs и многие другие, включая FAT, NTFS.
  • Жёсткая иерархия процессов (прародитель — init/systemd).

45Командные интерфейсы Linux. Командная оболочка Bash

Bash (Bourne Again Shell) — наиболее распространённая командная оболочка Linux. Командный интерпретатор читает и исполняет команды.

Bash поддерживает: переменные, условия (if), циклы (for, while), функции, скрипты (.sh файлы), перенаправление ввода-вывода (>, |), подстановку команд.

Часть команд: ls (список файлов), cd (смена каталога), mkdir/rm/cp/mv (работа с файлами), chmod (права), ps/kill (процессы).

46Процессы в ОС Linux. Атрибуты процессов

Каждый процесс имеет PID. Иерархия: каждый процесс создаётся через fork() и является дочерним по отношению к родителю. Прародитель — init (PID=1).

Атрибуты процесса: PID, PPID (PID родителя), состояние (running, sleeping, zombie…), приоритет (nice-значение), пользователь-владелец, используемая память и CPU.

Команды управления: ps (список процессов), pstree (дерево процессов), kill PID (завершить процесс), killall имя (завершить по имени). Сигналы: 9 (KILL — немедленно), 15 (TERM — нормально), 1 (HUP — остановить).

Три состояния процесса: выполнение, готовность, ожидание.

47Управление доступом в ОС Linux. Пользователи и группы. Администрирование

В Linux «делать всё» может только root (id=0). Остальные пользователи ограничены.

sudo команда — выполнить от имени root (пользователь должен быть в группе sudo/admin).
su имя — переключиться в оболочку другого пользователя.

Управление пользователями:
useradd -m имя — создать пользователя с домашней папкой.
passwd имя — установить пароль.
userdel имя — удалить пользователя.
usermod -aG sudo имя — добавить в группу sudo.

Права на файлы (chmod):
r=4 (чтение), w=2 (запись), x=1 (исполнение). Три группы: владелец (u), группа (g), остальные (o).
chmod 754 файл — владелец: все права (7), группа: чтение+исполнение (5), остальные: только чтение (4).
chown пользователь файл — сменить владельца (только root).

48Управление файлами и каталогами в ОС Linux. Команды

Основные команды: ls — список файлов; cd путь — переход в каталог; mkdir имя — создать папку; rmdir имя / rm -r имя — удалить папку; rm файл — удалить файл; cp источник цель — копировать; mv источник цель — переместить/переименовать; ln — создать ссылку; touch файл — создать пустой файл; chmod — изменить права доступа.

В Linux каталоги образуют дерево с корнем /. Домашние папки пользователей — /home/имя. Системные файлы — /etc, /bin, /usr и т.д.

49Операционные системы для мобильных устройств

Особенности ОС для мобильных: ограниченная память и мощность процессора; маленький экран; поддержка мультимедиа и сетевых технологий; различные форматы файлов.

  • Android (Google) — открытый исходный код, на базе ядра Linux. Наиболее распространён. Достоинства: гибкость, магазин приложений, многозадачность. Недостатки: фрагментация версий, расход батареи.
  • iOS (Apple) — закрытая ОС только для устройств Apple. Достоинства: безопасность, высокая скорость, качество приложений. Недостатки: закрытость, цена.
  • Windows Phone (Microsoft) — плавный интерфейс, интеграция с сервисами Microsoft. Недостаток: мало приложений. Фактически прекращена.
  • Bada (Samsung), BlackBerry OS, MeeGo — нишевые/устаревшие.

50Облачные вычисления

Облачные вычисления — модель предоставления вычислительных ресурсов как сервисов через Интернет: пользователю нужен только браузер, ПО и данные хранятся в центрах обработки данных (ЦОД).

Уровни архитектуры: клиент → сервисы → приложения → платформа → память → инфраструктура.

Модели:

  • IaaS — инфраструктура как сервис (виртуальные машины).
  • PaaS — платформа как сервис (среда разработки).
  • SaaS — программное обеспечение как сервис (Office 365, Google Docs).

Пример ОС: Windows Azure — облачная платформа Microsoft. Стандарты: HTTP, XMPP, HTML5, AJAX, XML.

Недостаток: полная зависимость от облака (потеря связи = потеря доступа к данным и программам).

51Перспективы развития операционных систем

  • Интеграция ОС (общие ядра для разных устройств).
  • Повышение надёжности, безопасности, отказоустойчивости.
  • Развитие виртуализации (запуск любых приложений в любой среде).
  • Сближение ОС для ПК и мобильных устройств.
  • Перенос ОС и инструментов в облако.
  • Развитие проектов с открытым исходным кодом.
  • Управляемый код (managed code) для безопасности.
  • Дальнейшая интеграция ОС и сетей.

52Безопасность информационных систем

Информационная безопасность — состояние защищённости информационной среды. Три ключевых свойства:

  • Конфиденциальность — доступ только авторизованным пользователям.
  • Целостность — достоверность и полнота информации.
  • Доступность — авторизованные пользователи получают доступ когда нужно.

Угрозы: природные/физические; непреднамеренные (ошибки пользователей); намеренные (атаки).

Механизмы защиты: кодирование (шифрование), сокрытие местоположения объектов, инкапсуляция. Средства: антивирусы, брандмауэры (межсетевые экраны), системы обнаружения вторжений (IDS/IPS), криптография, системы аутентификации (пароль, сертификат, биометрия), резервное копирование.

Криптография: открытый текст шифруется ключом в шифртекст. Только владелец ключа может расшифровать. Стойкость системы — степень её невозможности взломать.

53Меры обеспечения безопасности программного обеспечения

Защита ПО — комплекс мер против несанкционированного использования, копирования, распространения, изменения.

Локальная защита: требование серийного номера при установке/запуске.

Сетевая защита: проверка, не запущены ли две копии программы с одним ключом в сети.

Глобальная защита: программа проверяет ключ через центральный сервер (онлайн-игры, антивирусы).

Электронный ключ (донгл) — аппаратный USB/LPT-ключ с зашифрованными данными; без него программа не работает.

Привязка к компьютеру (активация): программа вычисляет уникальный код оборудования (BIOS, серийный номер диска) и привязывает лицензию к этой конкретной машине (пример: активация Windows).

Правовая защита: авторское право (компьютерные программы приравнены к литературным произведениям, гл. 70 ГК РФ); административная (КоАП, ст. 13.12) и уголовная (гл. 28 УК РФ) ответственность за нарушения.

54Виды угроз безопасности программного обеспечения

По происхождению:

  • Природные и физические — стихийные бедствия, пожары, перебои питания.
  • Непреднамеренные — ошибки пользователей, случайное удаление данных.
  • Намеренные — целенаправленные атаки.

По типу атак:

  • Вредоносные программы — вирусы, трояны, ransomware, spyware.
  • Инсайдерские атаки — хищение данных сотрудником с легальным доступом.
  • Внешние атаки — несанкционированный доступ в сеть извне (хакерские атаки).
  • Социальная инженерия — обман пользователей (фишинг).

Средства защиты: антивирусы; организационные меры (обучение пользователей); технические меры (брандмауэры, IDS/IPS, шифрование, разграничение прав доступа, резервное копирование).
